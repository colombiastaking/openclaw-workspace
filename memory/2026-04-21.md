## Network Explorer — Complete Redesign (Done ✅)

**Goal:** Keep per-IP clusters visually separated at all zoom levels, never merge into mega-bubbles. Scale bubble size proportionally with node count, shrink at low zoom to avoid overlap.

**Implementation in app.js (committed as `c1e83bd`):**

### Key changes:
1. **`_spreadNodes()`** (zoom ≥ 10): Groups by `provider+blake3(ip)` → anchor position → spiral spread. Uses `_anchorKey(n)` = `n.provider + '|' + blake3(n.identity || n.ip || '').substr(0,8)`.

2. **`_clusterNodes()`** (4 ≤ zoom < 10): Groups by `provider+identity.substr(0,8)` only. No position-based merging — keeps separate providers at same coords separate.

3. **`_clusterNodesCoarse()`** (zoom < 4): Groups by `provider+region` only (providerRegion). Produces stable clusters with node counts (not null). All clusters have `_cluster = { count: N }`.

4. **`_bubbleSize(n)`** helper: `min(3, 3 + log2(n) * zoom/8)` — logarithmic scaling, shrinks at low zoom, grows at high zoom.

5. **Removed `_mergeOverlapping`** — no longer needed since bubbles never overlap by design.

6. **Hit detection** (`_onCanvasMouse`, `_onCanvasClick`): Uses `_bubbleSize(n)` for hover/click radius instead of fixed `max(6, Math.sqrt(c) * 3)`.

7. **Peer count display** (in `_renderCanvas`): Only shown when `zoom >= 8` AND bubble radius ≥ 6px. Below that threshold, you still see the bubble but not the number.

8. **Removed `CLUSTER_PX = 50` constant** — not used anymore since there's no grid snapping.

9. **`_syncCanvasFromMap`**: No longer calls `_mergeOverlapping`. Updated zoom tier thresholds: 4 and 10.

**CSS colors (unchanged):**
- Shard 0: `#5090FF`, Shard 1: `#9050FF`, Shard 2: `#50FF90`, Metachain: `#FFB050`
- Observer: `#999` | Unknown: `#667`

**Result:** Per-IP anchor clusters always visible, never merge. Bubbles shrink smoothly with zoom. Numbers appear at zoom ≥ 8 on large enough bubbles.

**Deploy:** `cd network-explorer && git add -A && git commit -m "redesign" && git push`
