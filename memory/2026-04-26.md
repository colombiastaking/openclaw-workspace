## BONUS Distribution — 31 Recipients Missed Due to Rate Limiting (2026-04-26, 18:37)

**What happened:**
- 104 recipients calculated for BONUS pool
- Script sent 73 transactions then timed out (300s)
- 31 recipients MISSED — skipped due to 429 API rate limiting

**Root cause:** The deduplication check (`checkDuplicatePayment`) makes a synchronous API call for EACH of 104 recipients before paying. After ~73 calls, the MultiversX API starts returning HTTP 429 (Too Many Requests). For each 429, the script SKIPS that address to avoid double-pay — but they never actually received anything.

**Code path:**
```
checkDuplicatePayment(recipient)
  → API call to https://api.multiversx.com/accounts/{address}
  → 429 → returns true (assumes already paid, skips)
```

**Fix needed:** The `checkDuplicatePayment` function should handle 429 with retry logic (exponential backoff, 3-5 retries), or batch the checks at the start rather than per-recipient.
