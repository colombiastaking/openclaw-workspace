## COLS Distribution — Double-Payment Bug (Day 2)

### What happened today
- BONUS: 104 recipients, 24 double-sent, 29 never paid
- GOLD: 21 recipients, all correct
- DAO: failed (exit 1), 0 recipients
- **Total overpayment: ~2.42 COLS** (unrecoverable on-chain)
- **Total missing: ~6.07 COLS** (29 BONUS recipients never reached)

### Root cause confirmed
**MultiversX transfers API response format issue:**
- API returns: `[]` (plain array)
- Code expected: `{ data: [] }` (nested object)
- Code did: `data.data || []` → `undefined` → `[]`
- Result: on-chain dedup check always returned "no prior payment" → always sent to everyone

**Bug existed in both BONUS and GOLD sections.**

### Fix applied (commit a24cba3)
```js
// Before:
(data.data || [])

// After:
(Array.isArray(data) ? data : (data.data || []))
```

Applied to both BONUS and GOLD deduplication checks in `execute_distribution_fixed.cjs`.
