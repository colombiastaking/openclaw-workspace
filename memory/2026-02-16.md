## COLS Daily Distribution Sessions (CRITICAL)

### Session 7: Distribution Fixes ✅

**DAO Pool Transaction Fix:**
- Issue: First attempts failed with "sending value to non payable contract"
- Root Cause 1: Gas limit too low (600K instead of 10M)
- Root Cause 2: Function name not hex-encoded (`distribute` vs `64697374726962757465`)
- Root Cause 3: Entity address needed 32-byte hex, not string hex
- Solution: Correct transaction format with proper hex encoding

**Working DAO Transaction Format:**
```
ESDTTransfer@434f4c532d396439316237@<AMOUNT_HEX>@64697374726962757465@<ENTITY_32BYTE_HEX>
```
- Token hex: `434f4c532d396439316237` (COLS-9d91b7)
- Function hex: `64697374726962757465` (distribute)
- Entity hex: `00000000000000000500f5ae3a400dae272bd254689fd5a44f88e3f2949e5787`
- GasLimit: 10,000,000

**BONUS Pool Transaction Fix:**
- Issue: 62 transactions silently failed (no COLS received)
- Root Cause: Odd-length hex in amount field
- Example: `2fc1b5cdef91c` (13 chars) vs `02fc1b5cdef91c` (14 chars)
- Fix: Always pad hex with leading zero if odd length

**Critical Code Pattern:**
```javascript
function colsToHex(amount) {
  let hex = BigInt(Math.floor(amount * 1e18)).toString(16);
  if (hex.length % 2 !== 0) hex = '0' + hex;  // REQUIRED!
  return hex;
}
```

- All recipients now received correct amounts

- Server: LiteSpeed (not typical CDN)
