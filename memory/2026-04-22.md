## COLS Distribution — April 22 Incident & Fix

### What Happened (Incident Timeline)
- **165 ESDT TXs** from COLS wallet on April 22 (nonces 5125–5290)
- Normal expected: ~126 TXs (104 BONUS + 21 GOLD + 1 contract)
- **Root cause:** Script ran twice same day. No deduplication.
- **Double-paid:** 21 users received duplicate payments (amounts: 0.01–3.46 COLS extra per user)
- **Total overpayment:** ~19.4 COLS

### Trace (exact)
| Run | Nonces | TXs | Duration | Contents |
|-----|--------|-----|----------|----------|
| 1 | 5125–5166 | 42 | 119 min | Partial BONUS (failed at nonce 5167 — PeerMe DAO verification timeout) |
| 2 | 5168–5290 | 123 | 43 sec | Full BONUS (104) + GOLD (21) — script restart resending everything |

### Fix Applied (2026-04-22)
**File:** `DAPP-V3/scripts/execute_distribution_fixed.cjs`

**Logic (both BONUS and GOLD pools):**
```javascript
if (!resp.ok) {
  console.warn(`[COLSCHECK] API error for ${addr}, SKIPPING to avoid double-pay`);
  skipped++;
  continue; // ← Sebas wanted: don't send if API check fails
}
const txs = await resp.json();
const hasColsdToday = txs.some(tx => tx.timestamp >= startOfToday);
if (hasColsdToday) {
  console.log(`[COLSCHECK] ${addr} already received COLS today, SKIPPING`);
  skipped++;
  continue; // ← Already paid today, skip
}
// Normal send...
```

**Conservative by default:** If any API check fails (network error, rate limit, etc.) → **skip recipient entirely** (won't risk double-pay). Better to miss one payment than double-pay.
