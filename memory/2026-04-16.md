## Network Explorer — Major Bug Fixed

**Problem:** Map dropped from ~4,495 geo nodes to 1,468 after latest collector run.

**Root cause:** Path mismatch bug in `collector.js` STEP F. The code read `data/research-db.json` but the file lived at `server/data/research-db.json`. Result: STEP F found **0 entries** instead of 74 manual researched countries. 885 nodes that should have gotten real IPs → got none → no geo lookup → no lat/lng.

```bash

cd /home/raspberry/.openclaw/workspace/network-explorer
ln -sf server/data/research-db.json data/research-db.json
```

**Result after fix:** 4,370 nodes with lat/lng ✅

**Files involved:**
- `network-explorer/server/collector.js` — STEP F reads wrong path
- `network-explorer/data/research-db.json` — symlink created → `server/data/research-db.json`
- `network-explorer/scripts/research-sync.js` — merges `identity-country-map.json` into `research-db.json`

**Note:** The actual bug is in `collector.js` line reading `../data/research-db.json` instead of `server/data/research-db.json`. The symlink is a quick workaround. Could also fix the path in collector.js directly, but symlink is cleaner (doesn't modify the script path logic).
