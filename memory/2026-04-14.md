# 2026-04-14 Daily Notes

## Network Explorer — Sigma v2 + Mercator + Geo Circles (Major Fix)

**URL:** `https://colombia-staking.co/network/`

### Bug Fixed: Cluster Nodes Had Long Edges

**Root cause:** Two issues:
1. Anchor nodes had ±8px jitter — should be at exact geo coords
2. Cluster members (non-first nodes per IP) had `isGeo=false` → ForceAtlas2 wasn't freezing them → they drifted away → long edges to geo anchor

**Fix in `public/app.js` `_buildGraph()`:**
- Anchor: `jitterX/Y = 0` (zero displacement)
- All cluster members: `isGeo=true` (was only set for anchor)
- `isAnchor` attribute also set correctly (true for first node per IP)

**Result:** Only non-geo nodes (LAN IPs, dummy IPs, barycenter fallback) now move via ForceAtlas2. Geo clusters are frozen at exact positions.

### Previously Fixed (also today):
- `nodeClusterMap.has(bls)` → `nodeClusterMap.has(anchorIp)` (anchor was always unique per BLS)
- `posCache` restoring equirectangular over Mercator → geo nodes skip cache
- `_buildGraph()` and non-clustered geo placement using wrong equirectangular formula → replaced with `this._toSigma()`
- Sigma constructor stripped of v3-only options back to v2-compatible

### Deploy
```bash
cd ~/.openclaw/workspace/network-explorer && git add -A && git commit -m "fix: geo anchors at exact position, all members frozen" && git push
```
Live immediately at `https://colombia-staking.co/network/`
